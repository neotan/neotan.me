## 1. Homebrew

#### A. 中科大的源（推薦）

(1). [替換 homebrew-core源](https://lug.ustc.edu.cn/wiki/mirrors/help/homebrew-core.git)

運行：
```bash
cd "$(brew --repo)/Library/Taps/homebrew/homebrew-core"
git remote set-url origin git://mirrors.ustc.edu.cn/homebrew-core.git
```


(2). [替換 Homebrew Bottles源](https://lug.ustc.edu.cn/wiki/mirrors/help/homebrew-bottles)

```bash
echo 'export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles' >> ~/.bash_profile
source ~/.bash_profile

brew update
```



#### B. 清華的源

(1). [替換 formula 索引的鏡像（即 brew update 時所更新內容）](https://mirrors.tuna.tsinghua.edu.cn/help/homebrew/)

運行：
```bash
cd "$(brew --repo)"
git remote set-url origin https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/brew.git

cd "$(brew --repo)/Library/Taps/homebrew/homebrew-core"
git remote set-url origin https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/homebrew-core.git

brew update
```

(2). [替換 Homebrew 二進制預編譯包的鏡像](https://mirrors.tuna.tsinghua.edu.cn/help/homebrew-bottles/)

運行：
```bash
echo 'export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles' >> ~/.bash_profile
source ~/.bash_profile
```

**PS: 如果運行brew update的時候報了這段error，請使用中科大的源**

運行：
```bash
brew update
error: Packfile index for .git/objects/pack/pack-b2cf6673962afb44029dea036d48174efb361bc9.pack SHA1 mismatch
error: Unable to find a4d50c465d7398710f04022fafea080c7da83347 under https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/homebrew-core.git
Cannot obtain needed object a4d50c465d7398710f04022fafea080c7da83347
error: fetch failed.
```

## 2. Composer
來自[Packagist / Composer 中國全量鏡像](http://pkg.phpcomposer.com/)
也有兩種方式：

#### A. 修改 composer 的全局配置文件（推薦方式）

打開命令行窗口（windows用戶）或控制台（Linux、Mac 用戶）並執行如下命令：

```bash
composer config -g repo.packagist composer https://packagist.phpcomposer.com
```

#### B. 修改當前項目的 composer.json 配置文件：

打開命令行窗口（windows用戶）或控制台（Linux、Mac 用戶），進入你的項目的根目錄（也就是 composer.json 文件所在目錄），執行如下命令：

```bash
composer config repo.packagist composer https://packagist.phpcomposer.com
```

上述命令將會在當前項目中的 composer.json 文件的末尾自動添加鏡像的配置信息（你也可以自己手工添加）：

```json
"repositories": {
    "packagist": {
        "type": "composer",
        "url": "https://packagist.phpcomposer.com"
    }
}
```

## 3. npm
有永久和臨時兩種形式：

#### A. 通過 config 命令作全局永久替換

運行：
```bash
npm config set registry https://registry.npm.taobao.org 
npm info underscore 
```

如果上面配置正確，這個 npm info 命令會有字符串顯示，如:
```bash
...
dist:
 { shasum: '4f3fb53b106e6097fcf9cb4109f2a5e9bdfa5022',
   size: 34172,
   noattachment: false,
   tarball: 'http://registry.npm.taobao.org/underscore/download/underscore-1.8.3.tgz' },
directories: {},
publish_time: 1427988774520 }
```

#### B. 通過命令行臨時替換

運行：
```bash
npm --registry https://registry.npm.taobao.org info underscore 
```

### C. 另一種臨時方法
運行：
```bash
SASS_BINARY_SITE=https://npm.taobao.org/mirrors/node-sass/ npm install -g node-sass
```

## 4. apm (Atom editor)

[Atom 官方說apm是基于npm的](https://github.com/atom/apm#relation-to-npm))，所以可以使用上面淘寶的npm源。

編輯或新建文件`~/.atom/.apmrc`，加上這段：

```bash
registry=https://registry.npm.taobao.org/
strict-ssl=false
```

> 可選項，給apm加上代理：`apm config set proxy http://127.0.0.1:1080`


另外，Windows版的Atom會使用自帶的node.exe，而忽略你自行安裝的是哪個版本，為了統一node版本，可以將Atom自帶的node和npm都替換掉：

用Administrator模式啓動CMD，然後：
```bash
d:
cd "D:\Program Files\ATOM\resources\app\apm\bin"
node.exe -v
rename node.exe node.exe.original
mklink node.exe "D:\Program Files\node\node.exe"
node.exe -v
rename npm.cmd npm.cmd.original
mklink npm.cmd "D:\Program Files\node\npm.cmd"
cd ..
npm -v
```

安裝個插件試試，看看有沒有快些：

```bash
apm i sync-settings
```

另外，如果你用的是Windows Git Bash，裝好Atom後運行apm，python等命令是會提示`bash: apm: command not found`的，解決辦法很簡單，在Git Bash裏運行：

```bash
cd ~
echo "alias apm='winpty apm.cmd'" >> .bashrc
echo "alias python='winpty python.exe'" >> .bashrc
source .bashrc
```
